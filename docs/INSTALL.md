# Install

A GitHub profile README is just a repository named exactly the same as your
username, with a `README.md` at its root. GitHub detects that match and
renders it at the top of your profile page automatically.

## 1. Create the repository

- Create a new **public** repository on GitHub named exactly `AbhishekLim17`
  (must match your username exactly, case-sensitive).
- Do not initialize it with a README — you already have one.

## 2. Push this repository

From this folder:

```bash
git init
git add .
git commit -m "Initial profile README"
git branch -M main
git remote add origin https://github.com/AbhishekLim17/AbhishekLim17.git
git push -u origin main
```

## 3. Update the repository description

Go to **Settings** on the repository page and update the description to:

```
Founder · AI Engineer — building production AI systems
```

This text appears in search results, social previews, and the repository
header — it should match the identity presented in the README.

## 4. Verify asset paths render

GitHub renders relative image paths from the repository's default branch.
Once pushed, open `github.com/AbhishekLim17` and confirm:

- `assets/hero.svg` renders in the banner with animated neural network
- `assets/status-bar.svg` renders the terminal panel with fade-in text
- `assets/projects.svg` renders the two project cards
- `assets/architecture.svg` renders in the System Architecture section
- `assets/tech-grid.svg` renders the six technology category cards
- `assets/connect-bar.svg` renders the contact links bar
- The GitHub Analytics cards load (these are generated by third-party
  services and can take a few seconds on first load)
- The profile view counter badge loads at the bottom

## 5. Update contact details

Replace the contact information in two places:

**`assets/connect-bar.svg`** — update the display text for email and
LinkedIn (these are visual only, not clickable in the SVG):
```xml
<text ...>abhishek.limbachiya.edu</text>
<text ...>abhishek-limbachiya-18mar02</text>
```

**`README.md`** — update the actual clickable links in the Connect section:
```md
[`Email`](mailto:abhishek.limbachiya.edu@gmail.com)
[`LinkedIn`](https://www.linkedin.com/in/abhishek-limbachiya-18mar02/)
```

## 6. Keep it current

This README references specific, real projects (Haus of DADA, Magnetar
Global). As those change or new work replaces them, update the project cards
in `assets/projects.svg` and the **Currently Building** section rather than
letting it go stale — a profile README that hasn't changed in a year reads
as abandoned, regardless of how good the design is.
