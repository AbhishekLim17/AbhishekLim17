# Install

A GitHub profile README is just a repository named exactly the same as your
username, with a `README.md` at its root. GitHub detects that match and
renders it at the top of your profile page automatically.

## 1. Create the repository

- Create a new **public** repository on GitHub named exactly `AbhishekLim17`
  (must match your username exactly, case-sensitive).
- Do not initialize it with a README — you already have one.

## 2. Push this repository

From this folder:

```bash
git init
git add .
git commit -m "Initial profile README"
git branch -M main
git remote add origin https://github.com/AbhishekLim17/AbhishekLim17.git
git push -u origin main
```

## 3. Verify asset paths render

GitHub renders relative image paths from the repository's default branch.
Once pushed, open `github.com/AbhishekLim17` and confirm:

- `assets/hero.svg` renders in the banner
- `assets/architecture.svg` renders in the System Architecture section
- The GitHub Analytics cards load (these are generated by third-party
  services — `github-readme-stats` and `github-readme-streak-stats` — and
  can take a few seconds on first load)

## 4. Fill in contact details

Before publishing, replace the two placeholder lines in the **Connect**
section of `README.md`:

```md
`Email` → *add your contact email in README.md*
`LinkedIn` → *add your LinkedIn URL in README.md*
```

These were intentionally left as visible placeholders rather than invented
values — swap them for your real contact links.

## 5. Keep it current

This README references specific, real projects (Haus of DADA, Magnetar
Global). As those change or new work replaces them, update the **Currently
Building** section rather than letting it go stale — a profile README that
hasn't changed in a year reads as abandoned, regardless of how good the
design is.
